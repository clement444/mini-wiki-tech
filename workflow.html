Workflow Git

Un workflow Git simple consiste à utiliser des branches pour développer une fonctionnalité, puis à fusionner ces changements dans la branche principale.

Étapes

  • Créer une branche de fonctionnalité : git checkout -b feature.
  • Modifier les fichiers puis valider les changements : git add et git commit.
  • Ouvrir une Pull Request pour proposer la fusion.
  • Fusionner la branche après validation puis la supprimer.
